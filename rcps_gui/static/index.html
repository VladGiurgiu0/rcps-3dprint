Setup

The packing simulation uses PackingGeneration by Vasili Baranau (MIT license). It is not bundled with RCPS — it is cloned and compiled on your machine, or you point to a copy you already built.

Please cite when you use packings from it:
Baranau & Tallarek, Another resolution of the configurational entropy paradox as applied to hard spheres, J. Chem. Phys. 147, 224503 (2017) — doi:10.1063/1.4999483
or Baranau & Tallarek, Beyond Salsburg–Wood: glass equation of state for polydisperse hard spheres, AIP Advances 11, 035311 (2021) — doi:10.1063/5.0036411

Option A — build automatically

Requires git and a C++ compiler (macOS: xcode-select --install). Clones into packing-generation/ in the folder where rcps-gui was started and compiles the program sources directly (~1 min).

Pack

      
Force-biased start (-fba), then Lubachevsky–Stillinger
      densification (-ls, -lsgd).

      
Afterwards the diameters in packing.xyzd are rescaled
      to their true jammed values: the generator keeps the nominal
      diameter in the file and reports the real size as a ratio (its
      "inner diameter ratio"), per
      packing-generation issue #30. Without the
      rescale the file would describe overlapping spheres. Provenance goes
      to packing_meta.json.

Export

      
Full-resolution printable .3mf. A single tile, or a whole facility of
      interlocking tiles with an assembly map.

      
Mesher, bridges, printed-diameter choice and the advanced
      meshing options are taken from step 3 · Mesh — only the
      resolution and the facility grid are set here.
